# Агрегатор ведущих мероприятий России

Полнофункциональный прототип сайта-агрегатора ведущих мероприятий по всей России.

## 🎯 Функционал

### Роли пользователей:
1. **Ведущий** - регистрация, создание карточки, управление календарем, просмотр заявок
2. **Клиент** - поиск ведущих, бронирование дат, отслеживание заявок
3. **Админ** - модерация ведущих, управление категориями, просмотр всех заявок

### Основные возможности:
- Поиск ведущих с фильтрами (город, тип мероприятия, дата, цена)
- Календарь занятости ведущих (FullCalendar)
- Система бронирования с временной блокировкой дат
- Личные кабинеты для всех ролей
- Модерация контента

## 🛠️ Технологии

### Бэкенд:
- **Node.js** + **Express** - серверная часть
- **SQLite** - база данных (легко переключиться на PostgreSQL)
- **JWT** + **bcrypt** - аутентификация и авторизация

### Фронтенд:
- **HTML5** + **CSS3** + **JavaScript** (Vanilla)
- **Bootstrap 5** - стилизация
- **FullCalendar** - календарь занятости
- **Font Awesome** - иконки

### База данных:
- SQLite для простоты запуска
- 5 основных таблиц
- Легкая миграция на PostgreSQL

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
# Перейти в папку backend
cd backend

# Установить зависимости Node.js
npm install
```

### 2. Инициализация базы данных

```bash
# Создать базу данных и таблицы
node database/init.js
```

### 3. Запуск сервера

```bash
# Запустить сервер (порт 3000)
npm start
```

### 4. Открыть в браузере

Откройте файл: `frontend/index.html` в браузере

## 🔧 API Endpoints

### Аутентификация:
- `POST /api/auth/register` - регистрация
- `POST /api/auth/login` - вход
- `GET /api/auth/me` - информация о текущем пользователе

### Ведущие:
- `GET /api/performers` - список ведущих
- `GET /api/performers/:id` - информация о ведущем
- `POST /api/performers` - создание/обновление карточки
- `GET /api/performers/:id/calendar` - календарь ведущего

### Бронирования:
- `GET /api/bookings` - список заявок
- `POST /api/bookings` - создание заявки
- `PUT /api/bookings/:id` - обновление статуса

### Админ:
- `GET /api/admin/performers` - все ведущие для модерации
- `PUT /api/admin/performers/:id` - обновление статуса модерации
- `GET /api/admin/bookings` - все заявки

## 📅 Календарь занятости

### Особенности:
- Ведущий отмечает занятые/свободные дни
- Автоматическая блокировка дат с подтвержденными бронированиями
- Временная блокировка на 24 часа для новых заявок
- Клиент видит только свободные даты

### Цвета календаря:
- 🟢 Зеленый - свободно
- 🔴 Красный - занято
- 🟡 Желтый - ожидание подтверждения
- ⚫ Серый - заблокировано ведущим

## 🔐 Безопасность

- Хеширование паролей (bcrypt)
- JWT токены для аутентификации
- Валидация входных данных
- Защита от SQL-инъекций
- Проверка прав доступа

## 📊 База данных

### Таблицы:
1. **users** - пользователи (все роли)
2. **event_types** - типы мероприятий
3. **performers** - информация о ведущих
4. **performer_event_types** - связь ведущих и типов мероприятий
5. **calendar** - календарь занятости
6. **bookings** - заявки на бронирование

## ⚠️ Важные моменты

### Проверка доступности дат:
- При бронировании проверяется, что дата свободна
- Временная блокировка на 24 часа
- Автоматическое освобождение при отказе/истечении времени

### Конфликты дат:
- Нельзя забронировать уже занятую дату
- Ведущий не может отметить как свободную дату с подтвержденной заявкой
- Администратор видит все конфликты
